# 브레인시티 메디스파크 랜딩 페이지 기능정의서

## 1. 문서 목적

이 문서는 `브레인시티 메디스파크` 분양 랜딩 페이지 개발을 위한 기능 요구사항을 정리한 문서다.

목적은 아래와 같다.

- 상담 전환 구조 구현
- 고객 DB 수집 구조 명확화
- Google Apps Script 연동 기준 정리
- 운영 담당자가 바로 사용할 수 있는 수준으로 기능 정의

---

## 2. 기본 운영 정보

- 프로젝트명: 브레인시티 메디스파크 랜딩 페이지
- 대표번호: `010-6689-2348`
- 개발자명: `유희진`
- 저장 방식: `랜딩 폼 -> Google Apps Script -> Google Sheets`

---

## 3. 핵심 기능 요약

필수 기능은 아래 5가지다.

1. 상담하기 버튼 클릭 시 폼 노출
2. 이름 / 연락처 / 개인정보 동의 입력
3. 개발자 정보 자동 포함
4. Apps Script로 데이터 전송
5. 저장 완료 후 전화 연결 또는 완료 안내

---

## 4. 상담 버튼 기능

### 4-1. 버튼 종류

페이지 내 아래 버튼들은 모두 동일한 상담 폼을 호출한다.

- 히어로 영역 상담하기 버튼
- 중간 상담 유도 버튼
- 타입별 상담 버튼
- 하단 고정 상담 버튼
- 푸터 영역 상담 버튼

### 4-2. 공통 동작

버튼 클릭 시 아래 순서로 동작한다.

1. 상담 폼 팝업 또는 바텀시트 노출
2. 버튼 위치값 자동 세팅
3. 이름 / 연락처 입력 대기
4. 개인정보 동의 체크 확인
5. 제출 시 Apps Script 전송

---

## 5. 상담 폼 기능

### 5-1. 노출 방식

모바일:

- 하단에서 올라오는 바텀시트 형태 권장

데스크톱:

- 중앙 모달 팝업 형태 권장

### 5-2. 필드 구성

사용자 입력 필드:

- 이름
- 연락처
- 상담 목적
- 방문 희망일
- 희망 시간대
- 개인정보 수집 및 이용 동의 체크박스

숨김 필드:

- 개발자명: `유희진`
- 대표번호: `010-6689-2348`
- 유입페이지
- 버튼위치
- 문의유형
- 디바이스

### 5-3. 안내 문구

폼 상단 제목:

`빠른 상담 연결을 위해 간단한 정보만 남겨주세요`

보조 문구:

- `복잡한 절차 없이 상담 연결에 필요한 정보만 확인합니다.`
- `남겨주신 정보는 요청하신 상담 진행을 위해서만 사용됩니다.`
- `광고성 연락이 아닌 상담 요청 확인을 위한 최소 정보만 받습니다.`

### 5-4. 제출 버튼

권장 문구:

- `상담 전화 연결`

보조 문구:

- `대표번호 010-6689-2348`

## 6. 입력값 검증 규칙

### 이름

- 빈값 불가
- 최소 2자 이상 권장

### 연락처

- 빈값 불가
- 숫자만 저장
- 자릿수 검증 권장

### 개인정보 동의

- 동의 미체크 시 제출 불가

### 오류 메시지 예시

- `이름을 입력해 주세요.`
- `연락처를 정확히 입력해 주세요.`
- `개인정보 수집 및 이용에 동의해 주세요.`

```json
}
```

## 14. 개인정보 처리 관련 UI 요구사항

사용자 거부감을 줄이기 위해 아래 원칙을 반영한다.

- 입력 항목은 최소화한다.
- 광고성 연락이 아니라 상담 연결 목적임을 명확히 안내한다.
- 개인정보는 상담 용도로만 사용된다는 문구를 표시한다.

권장 문구:

- `입력하신 정보는 상담 요청 확인 및 연락을 위해서만 사용됩니다.`
- `상담 외 다른 용도로 사용하지 않습니다.`

---

## 15. 권장 페이지 내 상담 진입 지점

최소 3곳 이상 상담 진입 버튼을 둔다.

- 히어로 섹션
- 중간 설득 섹션
- 타입별 평면 섹션
- 하단 고정 버튼
- 푸터 영역

---

## 16. 향후 확장 기능

후속 운영을 고려해 아래 항목은 확장 가능하도록 설계한다.

- `utm_source`
- `utm_medium`
- `utm_campaign`
- 상담 희망 시간
- 관심 타입
- 방문 예약일

현재 1차 버전에서는 이름, 연락처 중심의 최소 구조로 시작한다.

- 상담 버튼 클릭 시 폼이 열려야 한다.
- 이름과 연락처를 반드시 입력받아야 한다.
- 개인정보 동의 없이는 제출할 수 없어야 한다.
- 개발자명 `유희진`이 자동 저장되어야 한다.
- 대표번호 `010-6689-2348`이 자동 저장되어야 한다.
- 데이터는 Google Apps Script를 통해 Google Sheets에 저장되어야 한다.
- 모바일에서는 저장 후 전화 연결이 가능해야 한다.
- 데스크톱에서는 저장 완료 메시지를 보여줘야 한다.
