## 1. 프로젝트 개요

이 문서는 `브레인시티 메디스파크 로제비앙 모아엘가` 분양 홍보 랜딩 페이지 제작을 위한 기획서다.

본 랜딩 페이지는 단순한 소개형 페이지가 아니라 아래 흐름을 목표로 하는 전환형 랜딩으로 설계한다.

1. 광고 및 외부 채널 유입
2. 입지와 상품성 설득
3. 상담하기 클릭 유도
4. 고객 DB 수집
5. Google Apps Script 연동
6. 구글시트 자동 저장
7. 전화상담 또는 후속 콜백 연결

대표번호는 `010-6689-2348`로 고정하며, 개발 담당자는 `유희진`으로 지정한다.

## 5. 상담 전환 구조

### 기본 흐름

1. 사용자가 `상담하기` 버튼 클릭
2. 이름과 연락처를 입력하는 간단한 상담 폼 노출
3. 거부감 없는 안내문 확인
4. 폼 제출
5. Google Apps Script로 데이터 전송
6. 구글시트 자동 저장
7. 모바일은 전화 연결, 데스크톱은 상담 접수 완료 안내

### 설계 원칙

- 바로 전화만 유도하지 않는다
- 사용자가 이름과 연락처를 먼저 남기게 한다
- 입력 거부반응이 없도록 최소 입력 구조로 설계한다
- 고객 DB에는 개발자 정보 `유희진`이 자동으로 포함되어야 한다

## 9. Google Apps Script 연동 기획

### 9-1. 저장 방식

랜딩 페이지 상담 폼에서 수집된 고객 정보는 `Google Apps Script`를 통해 지정된 `Google Sheets`에 자동 저장한다.

### 9-2. 저장 목적

- 상담 데이터 실시간 누적
- 상담 현황 관리
- 후속 응대 관리
- 개발자 정보와 함께 DB 보관

### 9-3. 기획 문구

`상담하기 버튼 클릭 시 간단한 고객정보 입력창이 열리며, 이름과 연락처 입력 후 제출하면 Google Apps Script를 통해 고객 DB가 구글시트에 자동 저장된다. 이때 저장 항목에는 상담 요청 고객 정보와 함께 개발 담당자 '유희진' 정보가 자동 기록되어 후속 응대 및 운영 관리에 활용된다.`
