# 브레인시티 메디스파크 상담 DB 구글시트 명세서

## 1. 문서 목적

이 문서는 랜딩 페이지에서 수집되는 상담 신청 데이터를 `Google Apps Script`를 통해 `Google Sheets`에 저장하기 위한 시트 구조 명세서다.

상담 데이터는 아래 목적을 위해 수집한다.

- 상담 신청 접수
- 전화상담 및 콜백 운영
- 개발 담당자 식별
- 유입 경로 분석
- 상담 처리 상태 관리

---

## 2. 기본 저장 원칙

- 사용자가 직접 입력하는 값은 최소화한다.
- 시스템 정보는 자동으로 기록한다.
- 개발자명 `유희진`은 숨김값으로 자동 저장한다.
- 대표번호 `010-6689-2348`은 고정값으로 저장한다.
- 날짜와 시간은 접수 시각 기준으로 자동 저장한다.

## 7. 사용자 입력 데이터 규칙

### 이름

- 공백 제거 후 저장 권장
- 최소 2자 이상 검증 권장

### 연락처

- 숫자만 추출해 저장 권장
- 예: `010-1234-5678` 입력 시 `01012345678`로 저장

### 개인정보동의여부

- 체크 시 `Y`
- 미체크 시 전송 차단

## 8. 버튼위치 값 정의

버튼 클릭 위치를 함께 저장하면 어떤 영역이 전환이 좋은지 분석할 수 있다.

권장 값:

- `hero_cta`
- `sticky_bottom_cta`
- `mid_form_cta`
- `type_section_cta`
- `footer_cta`
- `popup_cta`

---

## 9. 유입페이지 값 정의

유입페이지는 페이지 식별용으로 저장한다.

예시:

- `/braincity`
- `/braincity-medipark`
- `https://도메인주소/braincity-medipark`

실제 운영에서는 전체 URL 또는 고정 slug 둘 중 하나로 통일한다.

## 14. 운영 체크포인트

- 이름, 연락처, 동의여부 누락 시 저장하지 않는다.
- 중복 접수 여부를 후처리할 수 있도록 연락처는 통일된 형식으로 저장한다.
- 담당 개발자 정보는 사용자 입력이 아닌 시스템값으로 저장한다.
- 향후 광고 매체 식별이 필요하면 `utm_source`, `utm_campaign` 컬럼을 추가할 수 있다.
